## 2. Engineering Construct Model (sequence rationale)

The curriculum is organised into 4 constructs + GenAI, taught in this order:

1. **Engineering Concepts** — Design Patterns & Principles, Data Structures & Algorithms *(2 days)*
2. **Programming Languages** — PL/SQL (advanced SQL), TDD (JUnit5 + Mockito), SLF4J logging *(3 days)*
   *(Java + basic SQL are assumed from Upskilling)*
3. **Products & Frameworks** *(5 weeks)* — Spring Core & Maven, Spring Data JPA + Hibernate,
   Spring REST (Boot 3), Code Quality/SonarQube, Microservices (Spring Cloud), **Angular + Debugging**
4. **Platforms** *(1 week)* — Git, CI/CD & DevOps, Docker, Agile, Cloud (AWS)
5. **GenAI Fundamentals** — Prompt Engineering + GitHub Copilot

## 4. Submission Instructions (read carefully)

1. Mandatory hands-on per skill are listed in **`DN - Java FSE Mandatory hands-on detail.xlsx`**.
   The remaining exercises in each file are **additional / optional**.
2. Organise your solutions **week-wise** in folders.
3. Create a **public GitHub repository** in *your personal account*, upload your solution folder.
4. **Share the repo URL with your POC on demand.**
5. **Angular specifically:** all 10 hands-ons build **ONE** project (`student-course-portal`).
   Submit the whole project under `Angular_HandsOn/<YourName>/`, exclude `node_modules`
   (add to `.gitignore`). Do **not** create a separate project per Angular exercise.

## 6. Important Notes / Common Oversights

- **Mandatory ≠ everything.** Only the exercises flagged in the XLSX are required; but the KBA
  covers *all topics*, so still read the optional ones.
- **Angular is incremental** — skipping an early hands-on breaks later ones.
- **SonarQube, DevOps, Docker, Agile, Cloud, GenAI** have *no mandatory hands-on* in the XLSX
  but **are examinable** in the KBA. Don't ignore them.
- Country/Course domain recurs across Spring JPA + REST exercises — reuse your entities.
- Quizzes ("Check Your Understanding") are self-evaluation, not graded submissions — but good KBA prep.
